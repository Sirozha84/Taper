Как работает распознавание

Для понимания как исправить запись расскажу немного как работает распознавание.

При загрузке выборки определяются максимальная и минимальное положение, а ровно между ними определяется ось. «Куском» волны будет считаться часть, одна половина которой с одной стороны оси, другая - с другой. Длина волны и определяет бит: «единицы» длиньше «нолей» ровно в два раза. Если длина не подходит ни группе «0» ни группе «1» возникает сбой, который обозначается в подробном отчёте.

Продробный и очень подробный отчёт

0:07,373 - ---- Длина волны 21. Бит 0

0:07,374 - ---- Длина волны 42. Бит 1

0:07,375 - ---- Длина волны 21. Бит 0

0:07,375 - ---- Длина волны 42. Бит 1

0:07,376 - ---- Длина волны 42. Бит 1

0:07,377 - ---- Длина волны 42. Бит 1

0:07,378 - ---- Длина волны 21. Бит 0

0:07,378 - ---- Длина волны 21. Бит 0

Из этого следует на каком отрезке времени какая волна была обнаружена и какой, в связи с этим, бит был прочитан.

Кому этого не достаточно могут включить очень подробный отчёт, в котором помимо этого будет указываться поведение волны, в какое время с какой стороны оси она была.

0:03,674 - --->

0:03,674 - |

0:03,674 - |

0:03,674 -    |

0:03,674 -    |

0:03,674 -    |

0:03,674 - ---- Длина волны 6. Бит 0

0:03,674 - |

0:03,674 - |

0:03,674 - |

0:03,674 - |

0:03,674 - |

0:03,674 - |

0:03,674 -    |

0:03,674 -    |

Стрелкой обозначается направление изменения, в данном случае волна движется слева направо.

Зная что не нравится программе для «Чёткого» понимания можно исправить волну в любом звуковом редакторе.
